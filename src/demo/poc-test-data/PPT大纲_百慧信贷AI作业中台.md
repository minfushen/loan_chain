## 第一章：为什么要做这个中台（2页）

### 1-1 银行信贷作业的现实困境

**页面风格：** 左侧三个痛点卡片（浅蓝底），右侧一句话定位（大字）

**子内容标题：**
- 客户经理的三个"不知道"
  - 不知道今天先看哪一户
  - 不知道手里的证据够不够
  - 不知道AI给这个结论的依据是什么
- 传统作业模式的三个断点
  - 识别靠人脉 · 尽调靠经验 · 审批靠感觉
- 数字化现状的悖论
  - 银行有流水、有发票、有物流数据
  - 但没有把它们串起来变成判断的作业台
- 核心定位（大字强调）
  - **信贷作业的瓶颈不在数据，在于没有人帮客户经理把数据变成判断**

---

### 1-2 脱核链贷场景的特殊挑战

**页面风格：** 上方链条示意图（链主→二级→三级），下方左右对比（传统方式 vs 替代性证据）

**子内容标题：**
- 什么是脱核场景
  - 链主不确权、不出付款承诺
  - 三级供应商无法通过传统方式获得授信
- 传统方式的死路
  - 没有链主背书 → 无法授信 → 大量优质小微企业被排除在外
- 替代性证据的机会
  - 订单、物流、回款、发票四流数据已经存在
  - 缺的是识别、验证、推理的能力
- 市场空间
  - 新能源、制造业产业链三级供应商：银行小微信贷最大蓝海之一

## 第二章：解决什么业务问题（1页）

**页面风格：** 三列卡片（蓝色图标+标题+说明），底部一行系统边界说明

**子内容标题：**
- 问题一：谁该进池子
  - 从海量企业中识别有经营实质的主体
  - 不靠人脉靠数据，命中特征可解释
- 问题二：证据够不够
  - 四流交叉验证，替代性证据形成完整证据链
  - 每项核验结论有置信度、有来源、有冲突标注
- 问题三：为什么这样判断
  - 每笔授信建议有完整推理链
  - 满足银行合规审计的可解释性要求
- 系统边界（底部横条）
  - ✅ 做：预审推单 · 策略生成 · 规则配置 · 状态回流
  - ❌ 不做：清算放款 · 客户触达 · 数据采集 · 模型训练

## 第三章：核心方案是什么（2页）

### 3-1 八阶段AI作业导航

**页面风格：** 横向时间轴（8个节点），每个节点下方一句话说明，当前演示阶段高亮

**子内容标题：**
- 导航逻辑：按业务推进阶段排列，不按组织架构
- 八个阶段及核心问题
  - 今日工作台 → 今天先干什么、先看哪一户
  - 智能识别 → 谁该进池子、依据是什么
  - 智能尽调 → 证据是否充分、经营是否真实
  - 智能审批 → 为什么匹配这个产品、为什么能通过
  - 授信资产池 → 客户在哪个池子、下一步去哪
  - 智能监控 → 出了什么风险、该怎么处置
  - 智能经营 → 恢复、观察还是继续经营
  - 策略与配置 → 规则怎么配、数据从哪来
- 今日工作台设计理念
  - 打开首页立刻知道今天先干什么、先看哪一户，不是经营报表

### 3-2 两类智能体的架构与分工

**页面风格：** 上下两个泳道（决策智能体/审核智能体），中间触发箭头，右侧人机协作边界

**子内容标题：**
- 决策智能体（Decision Agent）
  - 职责：基于多源证据，自主推理输出授信建议
  - 六步工具调用链：任务分解 → 主体核验 → 财务分析 → 风险扫描 → 三流验证 → 反思输出
  - 自主性：半自主，结论需人工确认
- 审核智能体（Review Agent）
  - 职责：对决策智能体输出进行二次核验，处理异常与边界
  - 触发条件：置信度 < 85% 或命中异常规则
  - 四步处理链：异常分类 → 多假设验证 → 补充核实请求 → 双路径输出
  - 自主性：辅助型，不替代人工
- 人机协作边界
  - 自动决策：三流匹配率≥95% AND 置信度≥90% → 自动生成预授信建议
  - 必须人工：置信度<85% / 公私混用未解释 / 集中度>70% / 脱核场景
  - 智能体辅助人工：证据摘要卡 / 实时重算置信度 / 拒绝理由模板
- 为什么不用纯规则引擎（三个真实数据案例）
  - 案例一：公私混用（陈**，44次转账205万）
    - 规则：触发预警，案件标红
    - 智能体：72%可能是资金归集，只需补充一份身份证明
  - 案例二：Q3现金流为负（-10,809元）
    - 规则：现金流恶化，错误降级
    - 智能体：季末集中缴税+电费托收，正常季节性波动
  - 案例三：客户集中度72%
    - 规则：机械降额至96万
    - 智能体：产业链结构性绑定，维持120万+回款归集条件

### 4-2 智能审批：AI引导的四页作业台

- 补审作业

  - 收敛选择（产品匹配）/ 门槛推进（预审）/ 边界补强（补审）/ 最终裁决（审批摘要）

## 第七章：分阶段落地计划（1页）

**页面风格：** 三阶段横向时间轴，每阶段下方目标+关键里程碑

**子内容标题：**

### Phase 1 — POC验证（当前）
- 目标：核心链路可跑通，智能体推理逻辑可演示，获得种子客户反馈
- 已完成
  - 高保真可交互原型（v0.7.4）
  - 12类演示数据集，覆盖四流+财务+税务+工商+个人流水
  - 决策智能体+审核智能体架构设计与演示路径

- 待完成

### Phase 2 — 产品化（3-6个月）
- 目标：从单一场景扩展到多场景，形成可复用的SaaS产品内核
- 关键里程碑

  - 智能体能力持续迭代：基于多租户数据反馈优化推理准确率
  - 轻量接入方案：银行只需对接数据源，无需私有化部署
  - 定价模型：按授信案件数/按场景订阅/按智能体调用次数
