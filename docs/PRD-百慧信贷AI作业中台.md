## 1. 文档概述

### 1.1 目的

本文档定义「百慧」信贷 AI 作业中台的完整产品需求，涵盖产品定位、用户角色、功能规格、交互规范与数据模型。面向产品经理、前端工程师、交互设计师和业务方评审。

### 1.2 术语表

| 术语 | 定义 |
|------|------|
| 脱核链贷 | 不依赖链主企业直接确权，基于替代性经营证据为链上中小企业授信的融资模式 |
| AI 作业阶段 | 系统按业务推进顺序划分的 8 个一级模块，从识别到经营形成完整闭环 |
| 样本 | 贯穿全流程的企业实体，包含完整的供应链关系、经营数据和风险状态 |
| 证据四流 | 订单流、发票流、资金流、物流——用于交叉验证经营真实性的四类数据 |
| 浮动 AI Copilot | 右下角固定触发的 AI 引导面板，不参与布局网格，按模块切换上下文 |
| 预授信 | 系统基于证据和规则引擎自动生成的授信建议，需人工补审确认 |
| 链主 | 供应链核心企业，本系统中不要求其直接确权 |
| 关系强度 | 量化主体与产业链关联紧密程度的指标（0-100） |
| 证据覆盖度 | 订单/发票/回款/物流四流证据的完整程度（0-100%） |
| 集中度 | 最大客户占比，用于评估收入依赖风险 |
| 分层标签 | A可授信 / B可做但需处理 / C待观察 / D风险经营中——客群质量分级 |

## 2. 产品定位

### 2.1 一句话描述

百慧是面向银行信贷场景的 **AI 作业中台**，以 AI 作业阶段为导航主线，覆盖从智能识别、尽调、审批到监控、经营的全链路闭环。

### 2.2 产品本质

百慧不是传统业务后台，而是一个 **AI 作业指挥台**：

- 每个一级菜单对应一个 AI 辅助的作业阶段
- 用户按阶段推进业务，系统在每个阶段提供判断、证据和建议
- 系统承担预审推单、状态回流查看、策略生成、任务下发、规则配置等职能
- 不做清算、放款、审批建档、采集数据、训练模型等——由统一信贷系统、CRM、外数平台、模型工厂承担

### 2.3 系统边界（三不原则）

| 本系统做什么 | 本系统不做什么 | 由谁做 |
|-------------|--------------|--------|
| 预审推单、状态回流查看 | 清算、放款、审批建档 | 统一信贷系统 |
| 策略生成、任务下发 | 发短信、建商机、触达客户 | CRM / 企微 |
| 规则配置、能力调用编排 | 采集数据、训练模型 | 外数平台 / 模型工厂 |

### 2.4 目标用户

| 角色 | 职责 | 典型操作 |
|------|------|----------|
| 客户经理 | 日常作业主力 | 查看今日待办、推进审批、关注重点客户 |
| 审批人 | 风险把控决策 | 审批补审任务、确认授信额度、发起风险处置 |
| 风控专员 | 监控与规则维护 | 配置预警规则、查看风险信号、评估规则效果 |
| 经营管理员 | 客户分层与增收 | 管理恢复计划、配置经营模板、执行增收动作 |
| 系统管理员 | 后台配置 | 管理数据源、配置场景策略、维护报告模板 |

### 3.1 一级导航（8 个 AI 作业阶段）

```
今日工作台 → 智能识别 → 智能尽调 → 智能审批 → 授信资产池 → 智能监控 → 智能经营 → 策略与配置
```

### 3.3 导航层级规则

- **一级导航**：左侧固定 148px，按 AI 作业阶段排列，蓝底高亮当前项
- **二级导航**：SceneLayout 左侧边栏 208px，可折叠，支持分类分组
- **页面内导航**：面包屑（场景名 / 模块名）+ 页面标题 + 副标题

## 4. 功能需求详细规格

### 4.1 今日工作台（cockpit）

#### 4.1.1 今日总览（overview）

**目的**：打开首页立刻知道今天先干什么、先看哪一户、下一步点哪里。

**功能清单**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 个性化问候 | 根据时间显示「早安/午安/下午好/晚上好」，固定用户名 | P0 |
| 快速指标条 | 5 项指标：待补审/新增预警/恢复观察/待跟进客户/AI待确认建议，各配迷你趋势图 | P0 |
| 今日任务 | 高优先级/普通任务/我的跟进，按优先级分组展示，点击直达对应场景 | P0 |
| 当前重点客户 | SampleSwitcher 切换样本 + 四格信息卡（阶段/审批状态/风险状态/拟授信额度）+ 当前判断文案 + 下一步动作按钮 | P0 |
| AI 洞察提示条 | AiInsight 组件，根据高关系强度样本数和回款风险数动态切换 tone | P1 |
| 快捷入口 | 6 宫格：智能识别/智能审批/智能监控/智能经营/授信资产池/策略与配置，各显示统计数 | P0 |
| 样本分布图 | DonutChart 展示 ABCD 分层分布 | P1 |
| 演示控制 | ActionBar 按钮推进演示阶段 | P1 |

**AI Copilot 引导内容**：工作台总览使用说明，快捷导航至任务中心和 AI 建议。

#### 4.1.2 任务中心（task-center）

**目的**：统一汇集来自预警引擎、规则引擎和识别引擎的待办事项。

**功能清单**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 任务统计头 | 显示日期、当前页条数、涉及业务单元数 | P0 |
| 来源标签筛选 | 全部/补审/预警/催收/审批/跟进/恢复，圆角 pill 切换 | P0 |
| KPI 指标卡 | 4 张 MetricCard：补审队列/活跃预警/回款催收/恢复跟进 | P1 |
| 任务表格 | 列：来源(Badge) / 任务内容 / 数量 / 优先级 / 操作(处理按钮) | P0 |
| 底部快捷入口 | 3 张卡片：预警总览/处置作业/规则效果，链接到智能监控对应模块 | P1 |
| AI 提示条 | 动态显示高优先级任务数量，建议处理优先级 | P1 |

**数据模型**：`unifiedTaskRows`，每行包含 id/tag/title/count/priority/target/module/color/icon。

#### 4.1.3 消息通知（notifications）

**功能清单**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 未读统计 | Badge 显示未读数，全部标为已读按钮 | P0 |
| 通知列表 | 分类 Badge（审批/AI/预警/系统/恢复）+ 时间 + 标题 + 未读蓝点 | P0 |
| 单条已读 | 点击「标为已读」按钮 | P0 |
| 批量已读 | 一键全部标为已读 | P1 |
| AI 提示条 | 根据未读数切换 warning/success tone | P1 |

**数据模型**：`COCKPIT_NOTIFICATIONS` 静态数据，`readNotificationIds` 状态管理。

#### 4.1.4 我的关注（my-focus）

**功能清单**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| KPI 指标卡 | 4 张：星标/最近/AI推荐/全库样本 | P1 |
| 星标客户 | WorkbenchPanel 展示 `uiState.featured=true` 的样本 | P0 |
| 最近处理 | 按优先级和选中状态排序，取前 5 户 | P0 |
| AI 推荐关注 | 按置信度排序排除已星标，取前 4 户 | P1 |
| 客户行组件 | CockpitFocusCustomerRow：图标+名称+角色+状态Pill+箭头 | P0 |
| AI 提示条 | 星标/AI推荐数量统计 | P1 |

#### 4.1.5 AI 建议（ai-brief）

**功能清单**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 建议列表 | 按优先级(high/medium/low)和置信度排序的 AI 建议卡片 | P0 |
| 建议卡片 | 序号 + 企业名 + 优先级Badge + 置信度 + AI摘要 + 建议Agent + 建议动作 + 操作按钮 | P0 |
| 操作按钮 | 「查看客户」跳转智能识别 + 「相关审批」跳转智能审批 | P0 |
| AI 提示条 | 总建议数和置信度决策指引 | P1 |

**数据模型**：`aiBriefItems`，从 SAMPLES 映射并按优先级+置信度排序。

### 6.2 浮动 AI Copilot 规范

- **触发**：右下角固定 Brain 图标按钮（44px 圆形，primary 色）
- **展开**：右侧滑入面板（380px 宽），含半透明遮罩
- **内容**：标题 + 引导文案 + 快捷导航按钮列表
- **上下文切换**：每个二级模块自动切换对应引导内容
- **动画**：AnimatePresence + spring 物理动画

### 6.3 AiInsight 提示条规范

- 位于模块内容顶部（页头下方）
- 4 种 tone：info（蓝）/ warning（黄）/ success（绿）/ danger（红）
- 根据 业务数据 动态切换 tone 和文案
- 右侧可选 action 按钮（如「AI 建议」跳转）

### 6.4 多样本切换

- SampleSwitcher 组件全局可用
- 切换样本时自动同步：当前阶段、风险状态、恢复状态
- 顶栏显示当前样本名 + 审批状态 + Agent 辅助指示灯
- 底栏显示案例链名 + 样本名

### 6.5 演示流程控制

- ActionBar 组件提供阶段推进按钮
- 7 个阶段按固定顺序推进：ecosystem → identified → pre_credit → manual_review → approved → risk_alert → post_loan_recovery
- 阶段推进时自动导航到对应场景和模块
- 支持模拟风险事件和恢复经营
