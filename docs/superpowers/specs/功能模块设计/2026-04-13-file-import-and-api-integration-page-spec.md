# 文件导入与 API 接入页面规格 v1

## 文档目的

沉淀 `智能识别` 一级菜单下两个关键接入页面的结构化设计内容：

- `文件导入`
- `API 接入`

本文档覆盖：

- 卡片字段清单
- 按钮清单
- 页面文案稿 v1

本文档用于支持后续原型细化、前端实现与 PoC 演示口径统一。

---

## 1. 文件导入

### 1.1 页面定位

`文件导入` 不是单次上传页，而是面向 Excel / CSV 名单接入的批次管理台。

页面核心目标：

- 管理导入批次
- 查看字段校验状态
- 处理异常
- 生成样本
- 推进进入智能识别流程

---

### 1.2 卡片字段清单

#### A. 运行概览卡字段

每张概览卡建议统一 3 个字段：

- `状态名称`
- `批次数`
- `涉及企业数`

建议 5 张卡：

- `待校验批次`
- `待修正批次`
- `待生成样本`
- `待进入识别`
- `已完成批次`

#### B. 批次卡字段清单

每张批次卡建议放：

##### 基础信息

- `批次名称`
- `来源类型`：Excel / CSV
- `导入时间`
- `导入文件名`
- `企业总数`

##### 校验信息

- `通过校验数`
- `缺失字段数`
- `格式错误数`
- `重复企业数`
- `待人工确认数`

##### 生成信息

- `已生成样本数`
- `待生成样本数`
- `当前步骤`
- `当前状态`

##### 业务信息

- `建议场景`
- `下一步动作`

#### C. 详情抽屉字段清单

##### 页签：`批次信息`

- 批次名称
- 文件名
- 导入时间
- 企业总数
- 创建人
- 来源类型

##### 页签：`字段校验`

- 必填字段通过率
- 缺失字段明细
- 格式错误明细
- 重复企业明细

##### 页签：`样本生成`

- 生成样本总数
- 场景分布
- 阶段分布
- 推荐产品分布
- 风险初判分布

##### 页签：`流转记录`

- 导入完成时间
- 校验完成时间
- 样本生成时间
- 进入识别时间

## 2. API 接入

### 2.1 页面定位

`API 接入` 面向通过行内 ESB 总线接入企业名单批次的场景。

页面核心目标：

- 管理名单接入接口
- 查看接口运行状态
- 查看字段映射关系
- 追踪接入批次
- 处理失败与重试
- 将批次推进进入智能识别流程

---

### 2.2 卡片字段清单

#### A. 接入状态概览卡字段

建议 5 张卡：

- `已连接接口数`
- `运行中批次数`
- `失败批次数`
- `待重试批次数`
- `最近成功时间`

每张卡字段：

- `指标名`
- `值`
- `补充说明`

#### B. 接口卡字段清单

每个接口卡建议放：

##### 基础信息

- `接口名称`
- `接口类型`：ESB 推送 / ESB 拉取
- `接口编码`
- `当前状态`

##### 运行信息

- `最近调用时间`
- `成功率`
- `最近批次数`
- `平均处理时长`

##### 影响信息

- `影响模块`
- `当前异常数`
- `待处理批次数`

#### C. API 批次卡字段清单

每张运行中/失败批次卡建议放：

##### 基础信息

- `批次号`
- `来源接口`
- `接收时间`
- `企业总数`

##### 处理信息

- `已校验数`
- `异常数`
- `已生成样本数`
- `当前步骤`
- `当前状态`

##### 异常信息

- `失败原因`
- `影响业务动作`
- `是否可重试`

#### D. 字段映射卡字段清单

建议表格列：

- `上游字段名`
- `中台字段名`
- `字段说明`
- `是否必填`
- `转换规则`
- `当前状态`

示例：

- `ent_name` -> `企业名称`
- `credit_code` -> `统一社会信用代码`
- `legal_id` -> `法人身份证号码`

## 4. 两页统一提示文案建议

### 成功提示

- `批次接入成功`
- `字段校验完成`
- `样本生成完成`
- `已进入智能识别`

### 异常提示

- `存在必填字段缺失`
- `存在重复企业，请人工确认`
- `字段映射失败，请检查上游口径`
- `当前批次未生成样本，暂不能进入识别`

### 下一步提示

- `建议先处理异常字段，再继续流转`
- `该批次已具备进入识别条件`
- `建议先查看映射规则，再重新处理`

---

## 5. 文案使用原则

这两页的文案要始终围绕：

`批次、状态、异常、流转、下一步`

避免写成：

- 方案介绍
- 能力说明
- 技术架构讲解

这两页应保持明确的工作台语义。
