
# 经营总览页面规格 v1

## 一、文档目的

本文档用于明确 **智能经营 → 经营总览** 页面的功能定位、页面结构、字段定义、交互按钮及文案规范，为产品设计、原型开发及 PoC 演示提供统一依据。

---

## 二、页面定位

### 2.1 页面角色

`经营总览` 是智能经营模块的 **全局驾驶舱**，用于统一展示客户经营现状、价值分布、增长机会及流失风险，并为后续经营动作提供决策入口。

### 2.2 页面核心目标

* **全局洞察**：展示客户资产规模、价值结构及经营趋势。
* **机会识别**：发现可增收、可提升或可激活的客户群体。
* **风险预警**：识别沉默或流失客户，支持恢复经营。
* **行动指引**：为“客户分层”“经营动作”“客户恢复”提供下钻入口。
* **策略闭环**：通过模板与动作沉淀经营经验。

### 2.3 目标用户

| 用户角色    | 使用目标          |
| ------- | ------------- |
| 客户经理    | 识别经营机会并开展客户跟进 |
| 经营管理人员  | 监控整体经营成效与客户结构 |
| 风险/策略团队 | 判断客户变化趋势并优化策略 |
| 管理层     | 了解经营规模与增长潜力   |

---

## 三、页面结构设计

页面建议采用“驾驶舱式布局”，共分为五大区域：

1. **经营概览区**
2. **客户价值分层区**
3. **经营机会与恢复关注区**
4. **趋势分析区**
5. **AI经营洞察与行动建议区**

```
---------------------------------------------------------
| 页面标题：经营总览                                     |
---------------------------------------------------------
| 经营概览卡片（规模/增长/风险/机会）                   |
---------------------------------------------------------
| 客户价值分层 | 经营机会与恢复关注                     |
---------------------------------------------------------
| 经营趋势分析（新增、活跃、流失、转化）                |
---------------------------------------------------------
| AI经营洞察与行动建议                                  |
---------------------------------------------------------
```

## 四、功能模块与字段设计

### 4.1 经营概览区

#### 4.1.1 关键指标卡

| 指标名称    | 字段说明           |
| ------- | -------------- |
| 客户总数    | 纳入经营管理的客户数量    |
| 在营客户数   | 当前活跃且有业务往来的客户  |
| 高价值客户数  | 具有较高资产或贡献度的客户  |
| 可增收客户数  | 存在交叉销售或提额机会的客户 |
| 待恢复客户数  | 沉默或流失的客户       |
| 本期新增客户数 | 统计周期内新增客户      |

#### 4.1.2 字段属性

| 字段   | 类型  | 说明       |
| ---- | --- | -------- |
| 指标值  | 数值  | 当前统计结果   |
| 环比变化 | 百分比 | 与上一周期对比  |
| 趋势标识 | 图标  | 上升/下降/持平 |
| 下钻入口 | 链接  | 跳转至相关页面  |

### 4.2 客户价值分层区

| 类型     | 说明        |
| ------ | --------- |
| 提额机会   | 额度使用率高的客户 |
| 交叉销售机会 | 可匹配其他金融产品 |
| 续贷机会   | 临近到期客户    |
| 激活机会   | 沉默但具潜力客户  |

#### 4.3.2 恢复关注

| 类型     | 说明        |
| ------ | --------- |
| 沉默客户   | 一段时间未发生业务 |
| 流失风险客户 | 活跃度明显下降   |

#### 4.3.3 字段清单

| 字段      | 说明               |
| ------- | ---------------- |
| 客户名称    | 企业或客户名称          |
| 机会/风险类型 | 增收或恢复            |
| 资产规模    | 客户余额             |
| 最近活跃时间  | 最近交易或互动          |
| 推荐动作    | 系统建议             |
| 责任人     | 客户经理             |
| 下钻入口    | 跳转至“经营动作”或“客户恢复” |

### 4.4 经营趋势分析区

#### 4.4.1 展示维度

* 新增客户趋势
* 客户活跃度趋势
* 客户流失趋势
* 增收转化趋势
* 客户结构变化趋势

#### 4.4.2 字段清单

| 字段    | 说明        |
| ----- | --------- |
| 时间维度  | 日/周/月     |
| 指标名称  | 新增、活跃、流失等 |

| 同比/环比 | 变化趋势      |
| 图表类型  | 折线图/柱状图   |

---

### 4.5 AI经营洞察与行动建议区

#### 4.5.1 功能定位

通过AI分析客户行为与资产变化，自动生成经营洞察与行动建议，辅助用户制定经营策略。

#### 4.5.2 字段清单

| 字段     | 说明             |
| ------ | -------------- |
| 当前经营结论 | 对整体经营状态的总结     |
| 关键洞察   | 主要变化或异常        |
| 推荐行动   | 建议采取的经营动作      |
| 优先级    | 高/中/低          |
| 进入页面   | 客户分层/经营动作/客户恢复 |

#### 4.5.3 示例文案

* **当前经营结论**：整体客户结构稳定，但沉默客户占比有所上升。
* **关键洞察**：高价值客户中存在一定比例的续贷机会。
* **推荐行动**：优先针对沉默客户开展激活行动，并对高价值客户推进交叉销售。

## 六、页面文案规范

### 6.1 页面标题与副标题

* **页面标题**：经营总览
* **副标题**：统一洞察客户经营现状，识别增长机会与恢复对象，驱动经营策略落地。

### 6.2 空状态文案

| 场景    | 文案           |
| ----- | ------------ |
| 无客户数据 | 当前暂无客户经营数据。  |
| 无经营机会 | 当前未发现明显增收机会。 |
| 无恢复对象 | 当前暂无需恢复的客户。  |

### 6.3 成功与提示文案

| 类型 | 文案               |
| -- | ---------------- |
| 成功 | 经营数据已刷新          |
| 提示 | 建议优先关注高价值客户的续贷机会 |
| 警示 | 沉默客户数量上升，建议及时跟进  |

## 九、设计原则

1. **全局性**：作为经营模块的入口，需提供整体视角。
2. **可行动性**：不仅展示数据，还需引导用户采取行动。
3. **可下钻性**：支持快速跳转至相关功能页面。
4. **智能化**：通过AI洞察提供决策支持。
5. **可扩展性**：支持未来增加新的经营场景。

> **经营总览页面是智能经营模块的全局驾驶舱，旨在通过客户分层、机会识别与AI洞察，帮助用户快速理解经营现状并驱动后续经营动作，实现客户价值的持续提升。**
