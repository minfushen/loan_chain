# 前端文件改造任务表

## 文档目的

本任务表用于将“多样本脱核链贷业务作业台”的页面重构建议，落到具体前端文件层面，便于直接排期、拆分实现和跟踪进度。

本轮改造的核心目标有三项：

1. 统一全站 UI 风格、组件风格、版式结构，形成一致的 AI 资产作业台视觉语言。
2. 把系统从“单案例演示”升级为“多样本业务作业台”。
3. 按页面职责区分两类工作形态：
   - 流程化评估工作台
   - 长报告工作台

---

## 总体改造原则

### 页面结构原则

- 录入、校验、推进类页面采用“流程化评估工作台”。
- 分析、结论、确认、导出类页面采用“长报告工作台”。
- 首页只做全局控制台，不承载长流程或长报告。
- 每个页面只允许一个主结构，必要时允许一个辅助结构，不做平均铺开。

### 多样本改造原则

- 所有页面都区分“组合层”和“当前样本层”。
- 组合层负责全量池子、分布、规则效果、经营结果。
- 当前样本层负责当前选中样本的详情、证据、判断、动作。
- 不再允许 `CHAIN_LOAN_DEMO_CASE` 作为唯一主导对象长期存在于主工作区。

### 统一组件原则

- 所有页面统一引入样本切换条、当前样本摘要条、AI 洞察卡、动作建议卡、置信度卡等通用组件。
- 零散说明块逐步替换为统一的 `PanelCard / InsightCard / ActionSuggestionCard / ConfidenceCard`。

---

## 优先级定义

- `P0`：必须优先改，不改会导致多样本结构不成立或页面定位持续冲突。
- `P1`：建议本轮完成，不改会明显影响页面统一性和作业台表达。
- `P2`：可后续收口，不影响主链路，但会影响完整度和观感。

## 十一、最终验收标准

本轮改造完成后，应满足以下标准：

1. 所有主页面都具备统一的 UI 风格和组件语法。
2. 当前样本切换可驱动主工作区变化，而不是只切列表高亮。
3. 录入/评估类页面具备流程化评估工作台结构。
4. 分析/结论类页面具备长报告工作台结构。
5. 首页回归全局控制台，不再承担演示进度说明职责。
6. 合作方管理、授信资产池、产品审批、风险监控、贷后经营都具备明确的“作业台”属性。
7. 系统整体观感从“单案例演示原型”升级为“多样本 AI 资产作业台”。
