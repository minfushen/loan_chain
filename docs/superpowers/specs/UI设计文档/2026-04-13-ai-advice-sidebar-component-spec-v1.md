本文档聚焦：

## 一、组件定位

`AI 建议侧边栏` 是百慧全产品统一的 AI 决策组件。

它的核心职责不是展示长段说明，而是：

1. 在页面中持续保持 AI 建议存在感
2. 以轻量方式展示当前对象的一句话判断
3. 在需要时展开，承载完整判断与动作
4. 作为页面“判断 -> 操作”链路中的统一右侧助手

它不承担以下职责：

- 不承担聊天问答入口
- 不承载完整历史记录
- 不替代主页面详情区
- 不作为知识库配置入口

---

## 二、组件适用场景

该组件适合用于：

- 当前对象已明确的页面
- 页面需要 AI 判断与下一步建议的页面
- 页面右侧已有静态建议区，且阅读负担较重的页面

典型页面：

- 产品匹配
- 补审作业
- 审批摘要
- 材料解析
- 证据核验
- 智能推荐流
- 风险探针
- 客户恢复

不建议用于：

- 纯后台配置页
- 纯报表页
- 没有明确当前对象的空状态页

---

## 三、组件结构定义

组件建议分为 3 层结构：

## 1. 窄条外层

作用：

- 默认态展示
- 保持 AI 建议存在感

包含：

- AI 图标
- 状态点
- 一句话摘要
- 展开提示按钮

---

## 2. 展开内容层

作用：

- 展示完整 AI 建议内容

包含：

- Header
- 当前判断
- 关键依据
- 风险提示
- 下一步建议
- 动作区

---

## 3. 动作层

作用：

- 承接当前页面的主动作和次动作

包含：

- 主按钮
- 次按钮列表
- 可选的关闭 / 收起动作

---

## 四、组件内容结构规范

展开态内部建议固定为以下模块顺序：

1. `标题区`
2. `当前判断`
3. `关键依据`
4. `风险提示`
5. `下一步建议`
6. `动作区`

### 1. 标题区

建议包含：

- 组件标题：`AI 建议`
- 当前对象名（可选）
- 当前状态标签（可选）
- 收起按钮

---

### 2. 当前判断

内容要求：

- 一句话
- 明确结论
- 优先表达“能不能推进”

---

### 3. 关键依据

内容要求：

- 2 到 4 条
- 简洁
- 不能替代主页面的全部细节

表现建议：

- 短列表
- 标签点列

---

### 4. 风险提示

内容要求：

- 如有则显示
- 没有风险可不显示
- 必须足够具体

---

### 5. 下一步建议

内容要求：

- 必须与动作区一致
- 明确指出下一步去哪里

---

### 6. 动作区

建议包含：

- 主动作 1 个
- 次动作 1 到 2 个

---

## 五、组件状态机

组件建议至少支持以下 5 个状态。

## 1. `collapsed`

默认窄条态。

展示：

- AI 图标
- 一句话摘要
- 状态点

---

## 2. `previewing`

长悬停触发的预展开态。

特点：

- 准备展开
- 可在短延迟后进入 expanded

说明：

- 这是可选状态
- 如果实现复杂度过高，可跳过独立状态，直接由 hover delay 控制展开

---

## 3. `expanded`

展开阅读态。

展示完整内容。

---

## 4. `typing`

展开后内容逐字显示中的状态。

说明：

- 可以作为 `expanded` 的子状态
- 实现上也可用 `expanded + isTyping` 表达

---

## 5. `pinned`

用户主动点击展开后的固定展开态。

特点：

- 不因鼠标移出而自动关闭
- 必须手动收起或切换对象后自动重置

---

## 六、状态流转规则

推荐状态流转如下：

```text
collapsed
  -> hover delay -> previewing
  -> click -> pinned + expanded

previewing
  -> delay reached -> expanded + typing
  -> mouse leave -> collapsed

expanded + typing
  -> typing complete -> expanded
  -> click close -> collapsed

pinned + expanded
  -> click close -> collapsed
  -> switch entity -> typing restart
```

## 七、Props 结构建议

推荐组件 Props 分为 6 类。

## 1. 基础信息

```ts
title?: string
entityName?: string
summary: string
status?: {
  label: string
  tone: 'normal' | 'watch' | 'risk' | 'info' | 'muted'
}
```

作用：

- 驱动默认窄条态
- 提供标题和状态信息

---

## 2. 判断内容

```ts
judgment: string
basis: string[]
risk?: string
suggestion: string
```

作用：

- 驱动展开态主体内容

---

## 3. 动作配置

```ts
primaryAction?: {
  label: string
  onClick?: () => void
  tone?: 'brand' | 'success' | 'warning'
  disabled?: boolean
}

secondaryActions?: Array<{
  label: string
  onClick?: () => void
  disabled?: boolean
}>
```

作用：

- 统一动作区

---

## 4. 交互行为配置

```ts
expandMode?: 'click-only' | 'hover-and-click'
hoverDelayMs?: number
defaultCollapsed?: boolean
pinOnClick?: boolean
```

默认建议：

- `expandMode = 'hover-and-click'`
- `hoverDelayMs = 600`
- `defaultCollapsed = true`
- `pinOnClick = true`

---

## 5. 动画配置

```ts
typewriter?: boolean
typeSpeedMs?: number
replayOnEntityChange?: boolean
replayOnReopen?: boolean
```

默认建议：

- `typewriter = true`
- `typeSpeedMs = 14 ~ 22`
- `replayOnEntityChange = true`
- `replayOnReopen = false`

---

## 6. 页面上下文配置

```ts
entityId?: string
sceneId?: string
moduleId?: string
```

作用：

- 用于在切换当前对象时重置动画与内容
- 便于后续埋点和状态管理

---

## 八、默认行为建议

为保证全产品体验一致，建议组件默认行为如下：

### 默认收起

- 页面加载后默认显示窄条态

### 悬停预览

- 桌面端支持长悬停展开

### 点击固定

- 点击后展开并进入固定态

### 切换对象重置

- 当前对象切换时，摘要更新，展开内容重新触发逐字显示

### 再次打开不重播

- 同一对象重复展开时，默认不重新播放逐字动画

---

## 九、打字机动画规范

## 1. 动画目标

打字机效果的目标不是表演感，而是节奏感。

它应该让用户感受到：

- AI 正在解释当前对象
- 内容是逐步展开的
- 不会突然出现大段文本压迫视线

---

## 2. 动画适用内容

建议仅对以下内容启用逐字显示：

- 当前判断
- 下一步建议
- 简短风险提示

不建议对以下内容启用：

- basis 列表逐条全部逐字显示
- 按钮
- 标签
- 长表格

推荐表现：

- 判断先出现
- 依据快速淡入
- 风险和建议逐字补充

---

## 3. 动画时长建议

建议：

- 短句逐字显示总时长控制在 `400ms ~ 1200ms`
- 超过这个范围会影响效率

不建议：

- 模拟完整打字机长时间逐个字符输出

---

## 十、页面接入规范

## 1. 在 PageShell 中的接入位置

建议优先作为 `right` 区域的统一组件接入。

即：

- 左侧：对象列表
- 中部：详情区
- 右侧：AI 建议侧边栏窄条 / 展开态

---

## 2. 在页面状态中的接入

建议页面统一管理以下局部状态：

```ts
selectedEntityId
aiSidebarExpanded
aiSidebarPinned
```

如需更细，可增加：

```ts
aiSidebarTyping
```

---

## 3. 与当前对象联动

组件必须绑定当前对象。

即：

- 当前列表选中谁
- 当前详情展示谁
- AI 侧边栏就为谁生成摘要和建议

不允许：

- 页面已经切到 B 对象，但 AI 侧边栏还在说 A 对象

---

## 十一、不同页面的接入建议

## 1. 智能识别

建议：

- 摘要更强调推荐结论
- 主动作更强调进入筛选 / 尽调 / 候选池

---

## 2. 智能尽调

建议：

- 摘要更强调材料或证据状态
- 主动作更强调进入核验 / 生成报告

---

## 3. 智能审批

这是优先试点模块。

建议：

- 摘要更强调“是否可推进审批”
- 主动作更强调进入预审 / 补审 / 人工确认

---

## 4. 智能监控

建议：

- 摘要更强调风险等级与处置优先级
- 主动作更强调进入处置任务

---

## 5. 智能经营

建议：

- 摘要更强调恢复建议和经营动作
- 主动作更强调进入恢复经营 / 动作执行

---

## 十二、无障碍与体验要求

## 1. 键盘可达

组件必须支持：

- Tab 聚焦
- Enter / Space 展开
- Escape 收起

---

## 2. 减少动画干扰

建议预留：

- `prefers-reduced-motion` 兼容逻辑

当用户偏好减少动画时：

- 直接展示完整内容
- 不播放逐字效果

---

## 3. 文本可快速跳读

即使有打字机效果，也应允许：

- 用户点击“立即显示全部”
- 或在固定展开态直接完整显示

---

## 十三、不建议的实现方式

### 1. 不要把组件做成独立聊天面板

### 2. 不要让 hover 立即触发大面积展开

### 3. 不要在每次鼠标移出时强制关闭已 pin 的面板

### 4. 不要对所有文本都做逐字动画

### 5. 不要让组件脱离当前对象独立存在

---

## 十四、推荐的最小可落地版本

如果先做 v1，实现建议控制在最小闭环：

### 必做

- 默认窄条态
- 点击展开
- 展开态内容
- 主动作 / 次动作
- 当前对象切换联动

### 可选增强

- 长悬停展开
- 打字机动画
- pin 状态

### 后续增强

- 动画节奏配置
- 页面埋点
- 建议版本切换
- 历史建议回放

---

## 十五、一句话结论

`AI 建议侧边栏组件` 应被实现为百慧全产品统一的右侧决策组件：

**默认轻量、按需展开、对象绑定、节奏化解释、主动作稳定可达。**

它不是一个装饰性浮层，而是百慧从“页面展示系统”走向“AI 作业中台”的关键交互原语。
