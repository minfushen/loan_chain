# 百慧通用页面骨架模板

## 文档目的

为 `百慧` 当前和后续的大多数一级、二级页面提供一套统一可复用的页面骨架模板，用于指导信息架构、页面布局、交互节奏和组件使用的一致化设计。

本文档聚焦：

- 百慧页面的通用结构
- 各区域的职责定义
- 推荐组件组合
- 不同页面类型的变体模板
- 后续前端改造和页面设计时的统一准则

本文档适用于：

- 今日工作台
- 智能识别
- 智能尽调
- 智能审批
- 智能监控
- 智能经营
- 策略与配置

---

## 一、模板目标

通用页面骨架模板的目标，不是让所有页面长得完全一样，而是让它们在以下层面保持统一：

1. 用户一进入页面就知道当前位置与页面目的
2. 用户能快速建立“看什么、判断什么、下一步做什么”的节奏
3. 用户能在不同一级菜单之间获得一致的浏览和操作体验
4. 页面具备明显的主从关系，而不是信息平铺
5. 页面始终保留“AI 判断 + 下一步动作”的工作台特征

---

## 二、百慧页面的标准骨架

推荐百慧大多数页面统一采用以下 5 层骨架。

## 1. 顶部定位层

作用：

- 告诉用户当前在哪
- 告诉用户这个页面是干什么的
- 建立当前页面的全局上下文

建议包含：

- 面包屑
- 页面标题
- 页面副标题
- 页面级快捷动作

标准字段：

- `Breadcrumb`
- `Title`
- `Subtitle`
- `Primary Action`
- `Secondary Action`

示例：

- `智能审批 > 产品匹配 > 订单微贷`
- 页面标题：`产品匹配`
- 副标题：`根据主体特征、场景适配度与风险约束推荐可推进产品`

---

## 2. 全局摘要层

作用：

- 帮助用户快速获得全局状态
- 降低进入页面后的认知成本

建议包含：

- KPI Summary Bar
- 状态标签汇总
- 页面阶段提示

推荐指标数量：

- 3 到 5 个

不建议：

- 做成大面积复杂看板
- 使用过多同权重卡片

标准字段：

- `Metric Name`
- `Value`
- `Status / Delta`
- `Short Hint`

示例：

- 可推进主体数
- 待人工确认数
- 高风险项数
- 已完成数

---

## 3. 主工作区

作用：

- 承载页面核心任务
- 形成“对象选择 -> 详情判断 -> 推进行动”的主链路

推荐结构：

- 左侧：对象列表 / 批次泳道 / 任务队列
- 中部：详情工作区 / 主内容区
- 右侧：AI 判断与动作区

标准布局：

```text
左：列表 / 任务
中：详情 / 内容
右：判断 / 动作
```

### 3.1 左侧对象区

作用：

- 让用户快速定位当前处理对象
- 承担筛选、排序、切换上下文的功能

适合放：

- 批次列表
- 候选主体列表
- 报告列表
- 风险任务队列
- 经营客户列表

推荐组件：

- Master List
- Segmented Control
- Filter Bar
- Status Tags

---

### 3.2 中部内容区

作用：

- 展示当前对象的核心详情
- 承担页面最主要的信息阅读和判断工作

适合放：

- 详情卡
- 标签页
- 折叠面板
- 图谱 / 表格 / 证据列表
- 解析结果
- 产品说明

推荐组件：

- Tabs
- Accordion
- Drawer Trigger
- Timeline
- Detail Panels

---

### 3.3 右侧决策区

作用：

- 承接 AI 判断
- 明确当前结论
- 给出下一步动作

这是百慧区别于一般后台系统的关键区域。

建议固定包含：

- 当前判断
- 关键依据摘要
- 风险提示
- 下一步建议
- 主动作按钮

推荐组件：

- Decision Panel
- Sticky Action Card
- Status Tags

不建议：

- 把动作按钮散落在左中右多个区域
- 没有统一的“下一步动作”

---

## 4. 深度信息层

作用：

- 容纳低频但必要的补充信息
- 不抢占主工作区节奏

适合放：

- 历史记录
- 规则说明
- 版本记录
- 操作日志
- 详细证据

推荐组件：

- Drawer
- Accordion
- Timeline
- Secondary Tabs

---

## 5. 固定动作层

作用：

- 让当前页面最重要的动作始终可见
- 强化任务推进感

建议形式：

- 页面底部 Sticky Action Bar
- 右侧决策区的固定主按钮

推荐动作数量：

- 1 个主动作
- 1 到 2 个次动作

示例：

- `进入尽调`
- `进入预审`
- `补件作业`
- `人工确认`
- `生成报告`

---

## 三、通用页面骨架示意

```text
┌────────────────────────────────────────────────────────────┐
│ 面包屑 / 页面标题 / 页面副标题 / 快捷动作                  │
├────────────────────────────────────────────────────────────┤
│ KPI Summary Bar / 页面阶段提示                            │
├──────────────┬──────────────────────────────┬──────────────┤
│ 左侧对象区     │ 中部内容区                     │ 右侧决策区     │
│ 列表 / 队列    │ Tabs / Detail / Panels        │ AI / Actions   │
│ Filter / Tags │ Evidence / Graph / Timeline   │ Decision Panel │
├──────────────┴──────────────────────────────┴──────────────┤
│ 深度信息层：日志 / 历史 / 规则 / 低频说明                  │
├────────────────────────────────────────────────────────────┤
│ Sticky Action Bar                                          │
└────────────────────────────────────────────────────────────┘
```

## 四、推荐的页面区域职责表

| 区域 | 核心职责 | 不该承担的职责 |
| --- | --- | --- |
| 顶部定位层 | 说明页面身份与上下文 | 堆砌大量分析内容 |
| 全局摘要层 | 快速建立全局认知 | 代替主工作区 |
| 左侧对象区 | 列表、筛选、切换对象 | 展示深度细节 |
| 中部内容区 | 核心详情与阅读判断 | 分散关键动作 |
| 右侧决策区 | AI 判断与下一步动作 | 堆全部原始数据 |
| 深度信息层 | 历史、规则、日志、补充说明 | 抢占首屏主节奏 |
| 固定动作层 | 承接关键操作 | 承载过多复杂解释 |

## 五、推荐的组件使用规范

## 1. KPI Summary Bar

使用规则：

- 只放 3 到 5 个最关键指标
- 每个指标都应可快速扫描
- 不要放过多说明文字

适合：

- 批次数
- 待办数
- 异常数
- 可推进数

---

## 2. Tabs

使用规则：

- 只用于并列维度切换
- 不要把所有内容都做成 Tabs
- 1 级 Tabs 建议不超过 4 到 5 个

适合：

- 详情
- 依据
- 风险
- 历史

---

## 3. Accordion

使用规则：

- 用于低频、说明性、规则性内容
- 默认折叠
- 标题必须清晰表达折叠块内容

适合：

- 规则说明
- 异常详情
- 补充证据

---

## 4. Drawer

使用规则：

- 用于下钻而不是替代主页面
- 只承接深度信息或详情
- 打开后不能影响主流程判断

适合：

- 节点详情
- 原始证据
- 历史记录

---

## 5. Decision Panel

使用规则：

- 每页最多 1 个主判断
- 主判断必须对应主动作
- 不能只有结论没有依据

标准结构：

- 当前判断
- 关键依据
- 风险提示
- 下一步建议
- 主动作按钮

---

## 6. Sticky Action Bar

使用规则：

- 始终与当前页面主任务强相关
- 不要超过 3 个按钮
- 按钮优先级必须明确

建议结构：

- 主按钮 1 个
- 次按钮 1 到 2 个

---

## 六、百慧页面的 4 种通用模板变体

虽然页面可共用一套骨架，但不同页面类型仍应使用不同变体。

## 模板 A：列表决策型

适合：

- 候选资产列表
- 报告中心
- 风险任务
- 在营资产列表

结构：

- 左：对象列表
- 中：对象详情
- 右：决策面板

核心目标：

- 快速选对象
- 快速看状态
- 快速推进动作

---

## 模板 B：流程推进型

适合：

- 文件导入
- API 接入
- 材料解析
- 证据核验
- 预审与推单

结构：

- 顶部 Stepper
- 中部主流程内容
- 右侧动作区
- 底部固定动作栏

核心目标：

- 让用户明确知道当前流程阶段与下一步

---

## 模板 C：分析解释型

适合：

- 关系图谱
- 公私联动验证
- 风险探针
- 规则效果

结构：

- 顶部摘要条
- 中部分析画布 / 关系区
- 右侧解释与建议
- 下层深度证据

核心目标：

- 从分析结果快速过渡到解释与行动

---

## 模板 D：配置管理型

适合：

- 报告模板
- 数据源管理
- 产品配置
- 审批规则

结构：

- 左：配置项列表
- 中：配置主区
- 右：预览 / 影响说明

核心目标：

- 管理后台配置
- 保持结构化和可预览

---

## 七、不同一级菜单建议套用的骨架类型

| 一级菜单 | 推荐骨架类型 |
| --- | --- |
| 今日工作台 | 模板 A + 顶部工作首页强化 |
| 智能识别 | 模板 A / B / C 混用 |
| 智能尽调 | 模板 B / A 混用 |
| 智能审批 | 模板 B + A |
| 授信资产池 | 模板 A |
| 智能监控 | 模板 C + A |
| 智能经营 | 模板 A + C |
| 策略与配置 | 模板 D |

## 八、页面设计的统一节奏建议

无论使用哪种模板，百慧页面都建议遵循统一节奏：

### 第一步：先看摘要

用户进入页面后，先建立全局认知。

---

### 第二步：再选对象

用户明确当前处理的是谁、哪一批、哪一条任务。

---

### 第三步：看详情与依据

用户理解系统给出判断的原因。

---

### 第四步：看 AI 判断

用户获得总结、风险提示与建议。

---

### 第五步：做动作

用户通过主按钮推进流程。

---

## 九、前端实现时的统一要求

为避免后续落地时再次回到“卡片平铺”，建议统一遵循以下实现要求。

## 1. 视觉层级统一

- 页面标题 > KPI 摘要 > 主内容 > 补充内容
- 主按钮始终高于次按钮
- 不同卡片不要全部同尺寸同权重

---

## 2. 交互反馈统一

所有可点击对象建议具备：

- Hover 状态
- Focus 状态
- Active 状态
- 已选中状态

这样用户才能感知交互边界。

---

## 3. 主动作统一

每个页面都要能回答：

- 当前主动作是什么
- 它在哪
- 它是否始终可见

---

## 4. AI 判断区统一

建议所有页面使用统一决策区结构：

- 当前判断
- 关键依据
- 风险提示
- 下一步建议
- 主动作

---

## 十、一句话结论

`百慧通用页面骨架模板` 的核心，不是要求所有页面长得一样，而是让所有页面都遵循同一套工作台逻辑：

**先定位、再摘要、再看对象、再读依据、再看 AI 判断、最后推进动作。**

只有统一了这套骨架，百慧的界面体验才会真正从“原型页面集合”进化成“结构清晰、节奏统一、可推进的 AI 作业中台”。
