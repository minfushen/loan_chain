# 百慧页面设计体系总索引

## 文档目的

汇总 `百慧` 当前已经沉淀完成的页面设计相关文档，形成一份统一入口，便于后续产品设计、前端实现、交互优化和团队协同查阅。

本文档聚焦：

- 百慧页面设计体系的整体结构
- 各文档的定位与适用范围
- 推荐阅读顺序
- 后续页面设计与实现时的使用方式

## 一、页面设计体系的整体结构

当前百慧页面设计体系，建议按 4 个层次理解。

### 1. 总体优化层

回答：

- 当前页面为什么需要升级
- 系统应该从什么方向优化
- 应引入哪些结构型、流程型和决策型组件

对应文档：

- [百慧 UI/UX 系统化优化设计方案](/Users/admin/Documents/test007/硅基工作台/docs/superpowers/specs/2026-04-13-baihui-ui-ux-systematic-optimization-plan.md)

---

### 2. 通用骨架层

回答：

- 百慧页面应该长成什么结构
- 页面各区域分别承担什么职责
- 不同页面类型适合套用什么骨架

对应文档：

- [百慧通用页面骨架模板](/Users/admin/Documents/test007/硅基工作台/docs/superpowers/specs/2026-04-13-baihui-common-page-skeleton-template.md)

---

### 3. 通用规范层

回答：

- 间距怎么定
- 卡片怎么分层
- 按钮怎么定优先级
- 决策面板、状态标签、摘要条怎么统一

对应文档：

- [百慧页面设计规范 v1](/Users/admin/Documents/test007/硅基工作台/docs/superpowers/specs/2026-04-13-baihui-page-design-guidelines-v1.md)

---

### 4. 页面规格层

回答：

- 某个具体二级页面该如何设计
- 页面有哪些区块
- 卡片字段是什么
- 按钮是什么
- 页面文案如何写

对应文档：

- 智能识别组页面规格
- 智能尽调组页面规格
- 后续将继续扩展到智能审批、智能监控、智能经营

## 四、后续设计与实现时的使用方式

## 1. 设计新页面时

建议流程：

1. 先看 [百慧通用页面骨架模板](/Users/admin/Documents/test007/硅基工作台/docs/superpowers/specs/2026-04-13-baihui-common-page-skeleton-template.md)
2. 再看 [百慧页面设计规范 v1](/Users/admin/Documents/test007/硅基工作台/docs/superpowers/specs/2026-04-13-baihui-page-design-guidelines-v1.md)
3. 再写对应页面规格文档

---

## 2. 改造旧页面时

建议流程：

1. 先对照 [百慧 UI/UX 系统化优化设计方案](/Users/admin/Documents/test007/硅基工作台/docs/superpowers/specs/2026-04-13-baihui-ui-ux-systematic-optimization-plan.md) 判断当前页面的问题属于哪一类
2. 再用骨架模板重排页面结构
3. 最后用页面设计规范统一细节

## 六、后续建议的推进顺序

建议按以下顺序继续扩展设计体系：

1. 完成 `智能尽调` 组剩余 3 份页面规格
2. 再完成 `智能审批` 组页面规格
3. 然后补 `智能监控`
4. 最后补 `智能经营`

原因：

- 这条顺序与当前 PoC 主链最一致
- 也最符合百慧当前的产品价值验证重点
