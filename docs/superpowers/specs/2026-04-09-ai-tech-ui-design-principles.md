# AI 科技感 UI 设计原则清单

## 1. 文档信息

- 文档名称：AI 科技感 UI 设计原则清单
- 文档日期：2026-04-09
- 文档状态：Draft for Review
- 适用范围：`硅基工作台` 全站 UI 统一设计与后续 Agent 服务界面抽象

## 2. 目标

本清单用于指导当前原型从传统业务后台 UI 进一步升级为具备 AI 作业台气质的统一界面体系。

目标不是把界面做得更炫，而是让用户明确感受到：

1. 这是一个可信的业务系统
2. 这是一个正在实时分析和判断的智能系统
3. 这是一个可承接 Agent 服务抽象的控制台

## 3. 核心原则

### 3.1 科技感来自判断，不来自装饰

AI 科技感的核心不在于大面积玻璃、发光、赛博色彩，而在于系统是否在明确表达：

- 当前判断
- 命中规则
- 置信度
- 关键证据
- 下一步建议动作

应优先强化“推理感”“建议感”“正在执行”的体验，而不是优先堆叠视觉炫技。

### 3.2 基础骨架必须可信

无论 AI 科技感如何增强，全站基础仍需建立在可靠的业务系统骨架之上：

- 清晰的导航
- 稳定的页面分区
- 可读的表格和字段
- 明确的状态标签

AI 风格只能作为增强层，不能破坏金融业务系统的可信度和可读性。

### 3.3 AI 元素只出现在高价值区域

AI 科技感不应均匀铺满所有区域，而应集中在高价值模块：

- 全局命令输入
- AI 洞察卡
- 审批建议卡
- 风险探针卡
- 恢复经营建议卡
- Agent 能力卡
- 趋势预测图表

普通表格、规则列表、基础字段区应保持克制。

### 3.4 一页只保留一个科技焦点

每个页面最多只设置一个主要 AI 视觉焦点，例如：

- 审批页：证据交叉验证 + AI 审批建议
- 风险页：风险探针 + 额度联动
- 贷后页：恢复建议 + 恢复概率
- 客群页：识别结论 + 关系置信度

其余区域保持业务台稳定风格，避免整页多焦点抢夺注意力。

## 5. AI 卡片组件规范

### 5.1 AgentCapabilityCard

用途：表达一个 Agent 的职责与当前判断。

建议字段：

- Agent 名称
- 当前职责
- 输入来源
- 当前结论
- 置信度
- 建议动作

样式要求：

- 白底或浅蓝青渐变底
- 顶部带小状态点
- 右上角显示 `运行中 / 已完成 / 待确认`

适用页面：

- 产品与审批
- 风险监控
- 贷后经营
- 合作方管理

### 5.2 InsightCard

用途：展示一条由系统自动总结的关键洞察。

建议字段：

- 洞察标题
- 摘要说明
- 关键依据
- 推荐动作

样式要求：

- 视觉层级高于普通信息卡
- 可用冷色竖条、轻微边缘发光
- 不允许大面积炫彩背景

### 5.3 ConfidenceCard

用途：展示 AI 判断结果的可解释性和可信度。

建议字段：

- 当前结论
- 置信度百分比
- 命中规则数
- 证据覆盖度

样式要求：

- 使用细进度条
- 数值与规则命中数分层展示
- 不使用夸张仪表盘

### 5.4 ActionSuggestionCard

用途：表达系统建议的下一步动作。

建议字段：

- 建议动作
- 触发原因
- 紧急度
- 预计收益 / 风险改善

样式要求：

- 更像任务卡，而不是说明卡
- 支持主按钮和次按钮

### 5.5 SignalCard

用途：表达一个经营或风险信号。

建议字段：

- 信号名称
- 当前值
- 阈值
- 判断结果
- 最近变化

样式要求：

- 可带小趋势 sparkline
- 三态清晰：正常 / 观察 / 风险

### 5.6 TimelineInsightCard

用途：在时间轴中表达一个事件及其系统判断。

建议字段：

- 时间
- 事件
- 判断结果
- 后续动作

适用页面：

- 风险监控
- 贷后经营
- 合作方管理

## 6. 哪些页面该怎么体现科技感

### 6.1 首页 / 全局看板

科技感重点：

- 命令感
- 系统运行感

建议：

- 顶部搜索框升级为更像 command bar 的视觉
- 增加系统运行状态、最新分析时间
- 仅保留一个 AI 焦点区，例如“今日资产洞察”

避免：

- 全页每张卡片都使用 AI 风格

### 6.2 客群池

科技感重点：

- 识别推理
- 关系置信度

建议：

- 关系图谱页增加 AI 识别结论卡
- 标准小微页增加规则命中摘要
- 长尾场景页增加识别依据覆盖度

推荐组件：

- `ConfidenceCard`
- `InsightCard`
- `AgentCapabilityCard`

### 6.3 授信资产池

科技感重点：

- 转化判断
- 优先级调度

建议：

- 转化看板右侧增加“当前阻塞点洞察”
- 预授信池增加“推荐优先级”和“AI 建议产品”
- 补审队列增加“补审原因摘要卡”
- 在营资产增加“风险观察与机会发现”

推荐组件：

- `ActionSuggestionCard`
- `InsightCard`
- 小型预测趋势图

### 6.4 产品与审批

科技感重点：

- 可解释决策

建议：

- 三屏交叉验证之外，增加 AI 审批建议区
- 展示命中规则、证据覆盖、置信度
- 明确“为什么建议通过 / 为什么建议补充材料”

推荐组件：

- `AgentCapabilityCard`
- `ConfidenceCard`
- `InsightCard`

### 6.5 风险监控

科技感重点：

- 动态预警
- 风险探针

建议：

- 风险探针作为页面主焦点
- 图表增加阈值线、预测虚线、异常点
- 风险触发后增加明确的下一步处置建议

推荐组件：

- `SignalCard`
- 风险洞察卡
- 动态额度联动卡

### 6.6 贷后经营

科技感重点：

- 恢复判断
- 持续经营建议

建议：

- 恢复作业页增加“恢复条件达成概率”
- 增收动作页增加“系统建议优先顺序”
- 客户分层页增加“分层迁移预测”

推荐组件：

- `ActionSuggestionCard`
- `TimelineInsightCard`
- `ConfidenceCard`

### 6.7 合作方管理

科技感重点：

- 生态理解
- 数据接入成熟度

建议：

- 接入总览页增加“合作成熟度”“数据可用度”
- 数据来源页增加“可用性评分”
- 银行模板页增加“模板适配建议”

推荐组件：

- `AgentCapabilityCard`
- `InsightCard`
- 成熟度条 / 评分卡

## 7. 交互设计原则

### 7.1 命令交互是增强层，不是替代层

自然语言输入、全局指令入口非常适合后续 Agent 服务抽象，但在当前阶段应与稳定导航并存，而不是替代菜单、模块和页面结构。

### 7.2 动态数字必须有业务解释

动态数字不是为了动画，而是为了让用户感知变化。

推荐做法：

- 数字变化
- 趋势变化
- 一句 AI 自动摘要

例如：

- `回款周期 34 天`
- `较上周拉长 12%，接近预警阈值`

### 7.3 图表从“展示历史”升级为“辅助判断”

建议增强：

- 预测虚线
- 阈值线
- 风险区间底色
- 异常点标记
- 趋势摘要

不建议为了科技感加入大量无业务意义的流光动画。

## 8. Agent 服务抽象预留

后续若将能力抽象成 Agent 服务，UI 层应提前统一以下语法：

### 8.1 Agent 能力表达

统一展示：

- Agent 名称
- 当前职责
- 输入来源
- 当前结论
- 置信度
- 建议动作

### 8.2 推理结果表达

统一展示：

- 结论
- 命中规则
- 关键证据
- 风险或通过原因
- 建议动作

### 8.3 任务表达

统一任务语法：

- 待识别
- 待补审
- 待复核
- 待恢复
- 待跟进

## 9. 实施建议

建议按以下顺序落地：

1. 先定义设计 token
2. 再补 4-6 个 AI 通用卡片组件
3. 优先在以下三页落地科技感表达：
   - 产品与审批
   - 风险监控
   - 授信资产池
4. 最后统一到贷后经营和合作方管理

## 10. 终极判断标准

一个页面是否具备 AI 科技感，不看它是否足够炫，而看它是否同时满足：

1. 像一个可靠的业务系统
2. 能看出系统正在分析和判断
3. 能明确给出下一步动作建议

## 11. 推荐结论

本项目的 AI 科技感设计不应走“强赛博视觉”路线，而应采用以下原则：

**稳骨架，轻材质，强推理，弱炫技。**

这也是最适合当前银行普惠业务作业台、并能自然承接 Agent 服务抽象的 UI 方向。
