# AI 资产作业台 UI 统一改造方案

## 1. 文档信息

- 文档名称：AI 资产作业台 UI 统一改造方案
- 文档日期：2026-04-09
- 文档状态：Draft for Review
- 适用范围：当前 `硅基工作台` 原型全站 UI

## 2. 改造目标

当前原型已经完成了从“方案展示页”向“业务作业台”的第一轮收口，但仍存在以下问题：

1. 各一级页面的信息结构和视觉语言不统一。
2. 通用组件过少，页面大量依赖手写布局，导致风格漂移。
3. 顶部壳层、左侧模块区、主内容区仍然偏传统后台，缺少 AI 作业台的“系统中枢感”。
4. 页面虽然具备业务内容，但还没有形成后续可抽象为 Agent 服务的界面语法。

本次改造目标不是做“更炫”的界面，而是统一出一套可扩展、可复用、可持续演进的 UI 体系，让当前原型自然具备以下特征：

- 银行业务系统的可信度
- AI 作业台的控制感
- Agent 服务的模块化表达能力

## 3. 设计方向

### 3.1 设计定位

全站统一采用：

**可信业务骨架 + 轻科技交互层**

不做纯传统银行后台，也不做纯科幻展示页，而是在可靠、清晰、易理解的业务作业台基础上，加入轻量的 AI 控制台语言。

### 3.2 关键词

- 干净
- 冷静
- 精准
- 运转中
- 可指挥
- 可解释

### 3.3 避免方向

- 避免大面积营销渐变
- 避免通篇白卡片但没有层级
- 避免单页各写各的自由布局
- 避免过度发光和赛博装饰

## 4. 全站视觉语言

### 4.1 颜色体系

建议主色体系：

- 主背景：浅灰蓝白
- 主卡片：纯白
- 主强调色：冰蓝 / 电蓝
- 辅助强调色：青绿
- 风险色：冷红
- 警示色：琥珀橙
- 中性色：蓝灰

颜色角色分工：

- 蓝：当前选中、主链路、Agent 主能力
- 绿：通过、健康、恢复、稳定
- 橙：观察、人工处理中、待确认
- 红：风险、异常、额度收缩
- 灰：背景、说明、未激活

### 4.2 材质语言

材质不追求玻璃感，而采用：

- 白底卡片
- 轻描边
- 弱阴影
- 局部冷色辉光
- 微弱内发光边线

页面整体应有“系统在线运行”的质感，但不影响阅读。

### 4.3 字体与排版

- 页面标题：稳重、短、偏业务名词
- 模块标题：层级清晰，不使用口号式文案
- 说明文字：尽量短，更多使用字段表达
- 数据值：在卡片中应成为视觉重心

### 4.4 动效原则

- 页面切换保留轻过渡
- 数值变化采用短时平滑动画
- 风险与恢复场景允许局部强调动效
- 禁止全局炫技式运动

## 5. 全局壳层统一方案

### 5.1 顶部 Header

当前 header 仍偏传统后台，应统一成 AI 作业台头部。

建议结构：

1. 左侧品牌区
- 产品名
- 小型系统状态点

2. 中部主导航区
- 一级场景切换
- 当前场景高亮

3. 右侧控制区
- 命令式搜索
- 通知
- 用户身份
- 可选的 Agent 状态 badge

增强建议：

- 搜索框改为更像“命令搜索”
- 若演示模式开启，显示“资产引擎运行中 / Agent 正在辅助”的状态条

### 5.2 Footer / 系统状态条

底部状态条应从“弱提示”升级为“系统运行态指示”：

- 引擎状态
- 最新采集时间
- 当前案例模式
- 可选的规则版本 / 数据版本

## 6. 场景骨架统一方案

### 6.1 所有一级页面统一版式

所有场景页建议统一采用五段结构：

1. 页头摘要区
2. 当前案例或当前范围摘要区
3. 主工作区
4. 右侧关注 / Agent 建议区
5. 底部明细 / 时间轴 / 模板区

### 6.2 页头摘要区

页头只表达三件事：

- 当前页面名称
- 当前页面职责
- 当前模式标签

不再承担长段方案解释。

### 6.3 当前案例 / 当前范围摘要区

按页面类型展示：

- 若为案例页：展示当前企业、额度、状态、风险
- 若为组合页：展示当前资产范围、当前分层、当前关注

### 6.4 主工作区

主工作区建议固定为：

- 左侧：主分析面板
- 右侧：当前关注、待办、Agent 建议

### 6.5 底部区

底部统一承接：

- 记录流
- 模板
- 明细列表
- 时间轴

## 7. 通用组件体系

当前通用组件明显不足，需要扩成一个可复用的作业台组件集。

### 7.1 保留并重做

- `MetricCard`
- `FlowRow`

### 7.2 新增组件建议

#### 1. `PageHeader`

用途：统一页头信息

字段：

- 标题
- 副标题
- 标签组
- 可选右侧摘要

#### 2. `WorkbenchPanel`

用途：统一页面主卡片容器

特征：

- 带标题
- 带右上角操作位
- 可承载列表、图表、摘要

#### 3. `EntitySummaryCard`

用途：统一展示企业、链主、桥接节点、合作方等实体摘要

字段：

- 名称
- 角色
- 状态
- 关键值

#### 4. `StatusPill`

用途：统一所有状态标签

状态：

- 正常
- 观察
- 预警
- 已恢复
- 待补审
- 已审批

#### 5. `SignalCard`

用途：展示一个规则、一个风险信号或一个经营信号

字段：

- 信号名
- 当前值
- 阈值
- 结论

#### 6. `ActionQueueCard`

用途：展示当前待办、建议动作、排队任务

字段：

- 动作名称
- 来源
- 优先级
- SLA

#### 7. `AgentCapabilityCard`

用途：为后续 Agent 服务抽象预留视觉语法

字段：

- Agent 名称
- 当前职责
- 输入来源
- 当前结论
- 置信度

#### 8. `InsightStrip`

用途：展示一条高价值 AI 洞察或系统判断

#### 9. `TimelineRail`

用途：统一时间轴表达

#### 10. `MiniTrend`

用途：小型趋势展示

#### 11. `WorkbenchSplit`

用途：统一左右主次栏布局

## 8. 页面级统一模板

### 8.1 授信资产池

统一模板：

- 页头：资产转化状态
- 概览区：识别、预授信、补审、在营资产
- 主区：漏斗 / 分层 / 阻塞点
- 右栏：当前关注 / Agent 建议 / 下一动作
- 底部：样本列表 / 转化任务

### 8.2 贷后经营

统一模板：

- 页头：经营状态与恢复模式
- 概览区：额度、经营状态、阶段、风险
- 主区：经营健康面板 / 恢复条件 / 时间轴
- 右栏：提额、续贷、召回、交叉销售建议
- 底部：动作模板 / 经营轨迹 / 客户样本

### 8.3 合作方管理

统一模板：

- 页头：合作生态状态
- 概览区：链主、桥接方、内部数据方、外部数据方
- 主区：生态主链路图 / 数据接入情况
- 右栏：合作成熟度 / 当前缺口 / 下一步动作
- 底部：合作清单 / 模板适配 / 实施进度

### 8.4 风险监控

统一模板：

- 页头：风险总览
- 概览区：当前额度、风险级别、联动动作
- 主区：风险探针 / 指标变化 / 额度联动
- 右栏：建议动作 / 当前任务 / 置信度
- 底部：规则效果 / 历史趋势

### 8.5 产品与审批

统一模板：

- 页头：当前补审对象 / 作业模式
- 概览区：额度、产品、审批状态、证据完整度
- 主区：订单 / 物流 / 回款三屏交叉验证
- 右栏：规则命中 / 审批建议 / Agent 结论
- 底部：相关规则 / 历史补审样本

## 9. Agent 服务抽象预留

后续如将通用能力抽象为 Agent 服务，建议在 UI 层提前统一以下表达：

### 9.1 Agent 能力表达

统一采用卡片或条带表达：

- 关系识别 Agent
- 补审证据 Agent
- 风险探针 Agent
- 恢复经营 Agent

每个 Agent 卡需具备：

- 职责
- 当前输入
- 当前结论
- 置信度
- 建议动作

### 9.2 推理结果表达

不要只展示结论，要展示：

- 命中规则
- 关键证据
- 风险或通过原因
- 置信度

### 9.3 任务表达

统一任务语法：

- 待识别
- 待补审
- 待复核
- 待恢复
- 待跟进

## 12. 验收标准

### 12.1 视觉验收

- 所有页面具有统一视觉语法
- 所有一级场景像同一家产品输出
- 页面既有业务可信度，也有 AI 作业台气质

### 12.2 交互验收

- 页面层级清晰
- 关键信息不靠解释性长文支撑
- 当前关注、待办、建议动作具备统一表达

### 12.3 产品验收

- 页面能自然承接后续 Agent 服务抽象
- 页面结构具备模块化复用能力
- 不再保留明显方案页风格残留

## 13. 推荐结论

本项目下一步不应继续逐页单独修饰，而应先完成一次全站 UI 统一改造。

推荐执行路径：

1. 先重构通用组件与场景骨架
2. 再重构授信资产池、贷后经营、合作方管理三组重点页面
3. 最后统一风险、审批、客群和首页

这样能以最小返工成本，把当前原型提升为真正可承接 Agent 服务抽象的 AI 资产作业台。
